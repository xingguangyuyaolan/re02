# gazebo_pettingzoo_env.py 源码导读（当前版本）

对应文件：`src/scripts/gazebo_pettingzoo_env.py`

更新时间：2026-04-20

## 1. 文件职责

该文件将 Gazebo + ROS2 多无人机仿真封装为 PettingZoo ParallelEnv，提供训练所需接口：

1. `reset()`
2. `step(actions)`
3. 观测构建与奖励计算
4. 安全约束与终止判定

## 2. 观测与动作

### 2.1 动作

策略输出离散动作，环境执行时映射为 4 维连续控制。

### 2.2 观测

每个智能体观测由多部分组成：

1. 位姿/速度
2. 激光雷达
3. 覆盖相关特征
4. 局部覆盖图
5. 跨智能体相对信息

## 3. 回合逻辑

回合终止由以下条件驱动：

1. 达到最大步数
2. 所有 agent 进入 done

并在 `info` 中记录碰撞、出界、覆盖等任务状态。

## 4. 奖励与惩罚

当前环境主要包含：

1. 覆盖增益奖励
2. 重复覆盖惩罚
3. 重叠覆盖惩罚
4. 碰撞惩罚
5. 出界惩罚
6. 边界软惩罚
7. 高度安全惩罚
8. 生存奖励（可配置）

## 5. 安全机制

包含：

1. 启动宽限步（startup grace）
2. 碰撞冷却步（collision cooldown）
3. 碰撞是否终止可配置（`collision_terminal`）
4. 高度硬下限与最小高度约束

## 6. reset 稳定化机制

围绕长期训练稳定性实现了：

1. reset 超时保护
2. 传感数据可用性等待
3. 位置与布局验证
4. reset 失败回退和重试参数

这些参数可由配置下发，在训练和评测中保持一致。
