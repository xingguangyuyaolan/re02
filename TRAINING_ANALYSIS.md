# 训练分析与阶段结论（更新版）

更新时间：2026-04-20

## 1. 分析范围

本文记录当前项目在 Stage1 主线上的训练现状、问题定位与下一步动作，面向代码当前版本：

- 训练入口：`train.py`
- 课程脚本：`src/scripts/train_curriculum_p0.sh`
- 算法：`src/scripts/attention_qmix.py`
- 环境：`src/scripts/gazebo_pettingzoo_env.py`
- 评测：`evaluate.py`

## 2. 当前阶段结论

1. 训练链路与评测链路已打通，模型可稳定保存与复评。
2. 与早期基线相比，覆盖能力和全覆盖成功率已有明显提升。
3. 主要瓶颈仍是高碰撞率与后期策略退化（尤其在低 epsilon 段）。
4. 出界问题已较早期有所缓解，但仍是影响稳态表现的重要因素。

## 3. 已确认的关键修复

### 3.1 课程阶段写入目录修复

- 问题：Stage2 曾写入 Stage1 目录，导致指标污染。
- 修复：跨阶段 resume 时创建新 run 目录。
- 影响：阶段结果独立、可追溯。

### 3.2 epsilon 跨阶段继承修复

- 问题：Stage2 初始 epsilon 被 Stage1 checkpoint 覆盖。
- 修复：跨阶段只迁移网络权重，重置 episode/total_steps/epsilon。
- 影响：新阶段探索行为符合配置预期。

### 3.3 评测脚本参数同步修复

- 问题：评测未完整恢复训练结构与环境参数。
- 修复：`evaluate.py` 已恢复注意力配置和关键环境参数。
- 影响：评测结果与训练配置的一致性显著提高。

## 4. 当前风险点

1. 碰撞率依然偏高，说明局部安全策略仍不足。
2. 后期可能出现性能回落，提示奖励和探索节奏仍需细化。
3. 复杂场景迁移（Stage2/Stage3）仍需持续验证稳定性。

## 5. 下一步执行计划

1. Stage2 持续训练并固定评测节奏（每 N episode 评测一次）。
2. 做注意力消融：无注意力/自注意力/跨智能体注意力/全注意力。
3. 强化安全相关奖励项与边界先验，继续压低碰撞与出界。
4. 将阶段性最佳模型统一做跨场景评测，形成中后期论文图表集。

## 6. 建议的阶段验收指标

1. `full_coverage_success_rate` 持续提升且波动收敛。
2. `collision_rate`、`out_of_bounds_rate` 相对基线持续下降。
3. `coverage_rate_mean` 在不同阶段都保持增长趋势。
4. 同配置重复运行结果偏差可控（可复现）。
